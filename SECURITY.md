# Security Policy

## Supported Versions

Before the first stable release, only the latest release and current `main` receive security fixes. After 1.0, this table will be updated with an explicit support window.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's **Report a vulnerability** flow to create a private Security Advisory for this repository. Include:

- affected version or commit;
- impact and realistic attack scenario;
- reproduction steps or proof of concept;
- suggested remediation, if known;
- whether any secret or user data may have been exposed.

Maintainers will acknowledge a complete report as soon as practical, coordinate validation and remediation in the advisory's private fork, and publish the fix and advisory together.

Do not include live credentials, private customer payloads, or unrelated personal data in a report.
